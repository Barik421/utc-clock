# Тестування розширення перед публікацією

## 1. Підготовка локально

1. Відкрийте проєкт у VS Code: `/Users/brnv/Documents/UTC clock`
2. У терміналі виконайте:
   ```bash
   git pull
   ```
3. Переконайтесь, що немає незакомічених змін (або збережіть їх):
   ```bash
   git status
   ```

## 2. Встановлення в Chrome (Developer mode)

1. Відкрийте: `chrome://extensions`
2. Увімкніть `Developer mode`.
3. Натисніть `Load unpacked`.
4. Оберіть папку проєкту.
5. Закріпіть іконку розширення в toolbar.

## 3. Функціональний чекліст (обов'язково)

1. Popup відкривається без помилок.
2. Відображається UTC час у форматі `HH:MM:SS`.
3. Час змінюється щосекунди.
4. Відображається UTC дата.
5. Кнопка `Settings` відкриває/закриває налаштування.
6. Перемикання мови:
   - English -> усі підписи англійською.
   - Українська -> усі підписи українською.
7. Перезавантажте popup і перевірте, що обрана мова збереглась.
8. Перемикання теми:
   - Light тема застосовується коректно.
   - Dark тема застосовується коректно.
9. Перезавантажте popup і перевірте, що обрана тема збереглась.
10. Кнопка `Open UTC` відкриває зовнішню сторінку UTC у новій вкладці.
11. Перевірте, що іконка розширення видно в тулбарі Chrome.

## 4. Візуальний чекліст (UI)

1. Кольори відповідають заданому стилю: сірий фон + червоний акцент.
2. Текст читабельний у Light і Dark темі.
3. Кнопки мають достатній контраст і зрозумілий стан `active`.
4. Немає "з'їханої" верстки або обрізаного тексту.

## 5. Перевірка помилок

1. У `chrome://extensions` відкрийте `Errors` для цього extension.
2. Переконайтесь, що JavaScript помилок немає.
3. Якщо є помилки, виправте і натисніть `Reload` у `chrome://extensions`.

## 6. Підготовка до релізу в GitHub

Після успішного тесту виконайте:

```bash
cd "/Users/brnv/Documents/UTC clock"
git add .
git commit -m "chore: prepare extension for release"
git push
```

## 7. Пакування для Chrome Web Store

1. Створіть ZIP з файлів проєкту (без `.git`):
   ```bash
   cd "/Users/brnv/Documents/UTC clock"
   zip -r utc-clock-release.zip . -x "*.git*" "*.DS_Store"
   ```
2. Перевірте ZIP: всередині мають бути `manifest.json` і папка `src/`.
3. Завантажте ZIP у Chrome Developer Dashboard.

## 8. Перед натисканням "Publish"

1. Перевірте назву, опис, скріншоти, іконки.
2. Перевірте політики/конфіденційність (якщо потрібно).
3. Переконайтесь, що версія в `manifest.json` оновлена для нового релізу.
